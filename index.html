Semantic Web

Sofia Berdichevska, Franca Lenz, Max Kohla

Was ist Sematic Web?

  • Technoglogie, welche Daten Maschinenlesbarer macht
  • Fokussiert auf Bedeutung von Daten, für bessere Struktur
  • Verknüpfung und Integration von Daten aus verschiedenen Quellen

Ziele des Semantic Web

  • Verknüpfung und Austausch von Informationen zwischen Systemen.
  • Verbesserte Interoperabilität: Plattformübergreifende Datenintegration.
  • Automatisierte Entscheidungsfindung: Unterstützung von Anwendungen wie KI und personalisierten Diensten.

Technologien des Semantic Web

RDF (Resource Description Framework)

  • Informationen im Web klar und verständlich darstellen
  • in Form von Subjekt (wer/was), Prädikat (was wird gesagt) und Objekt (was genau) - Tripel gespeichert

    • Beispiel: „Das Gemälde (Subjekt) wurde gemalt von (Prädikat) Vincent Van Gogh (Objekt)“

  • verbindet verschiedene Daten im Web und macht sie für Maschinen lesbar
  • Vorteil: flexibel, kann mit verschiedenen Datentypen & Formaten erweitert werden

OWL (Web Ontology Language)

  • ermöglicht Erstellung von Ontologien (Modellen) die Begriffe und deren Beziehungen definieren
  • legt genau fest was ein Begriff bedeutet (z.B „Gemälde“)
  • beschreibt wie Begriffe miteinander verbunden sind (z.B "Gemälde" und Künstler“)
  • hilft Bedeutung und Semantik von Web-Inhalten besser zu verstehen

SPARQL (SPARQL Protocol and RDF Query Language)

  • Abfragesprache des Semantic Web
  • ermöglicht spezifische Fragen an RDF-Daten zu stellen
  • traditionelle Suchanfragen basieren auf Schlüsselwörtern
  • SPARQL berücksichtigt Bedeutung und Kontext von Daten - intelligente Suchanfragen
  • vielseite Abfragen (z.B Informationen über den Stil und die Epoche eines Gemäldes)

Rule Interchange Format (RIF)

  • Standard, der Regeln zwischen verschiedenen Regelsprachen übersetzt
  • Regeln - „WENN … DANN …“-Anweisungen mit denen Computer logische Schlüsse ziehen (z. B. „Wenn A ein Bruder ist, dann ist B die Schwester“).
  • RIF ermöglicht Regeln in verschiedene formale Sprachen zu übertragen, um sie universell nutzbar zu machen
  • verschiedene Dialekte, z. B. RIF-BLD (für Logikregeln) und RIF-PRD (für Produktionsregeln)

JSON-LD (JSON for Linking Data)

  • Erweiterung von JSON, um verknüpfte Daten darzustellen
  • fügt Bedeutung zu Daten hinzu ("Name: Max" wird zu "Person heißt Max")
  • macht Webdienste verständlicher für Maschinen
  • häufig in APIs und Webseiten genutzt

SHACL (Shapes Constraint Language)

  • Sprache, um Regeln für die Validierung von RDF-Daten zu definieren
  • überprüft, ob RDF-Daten bestimmten Anforderungen (sogenannten "Shapes") entsprechen
  • hilft sicherzustellen, dass Daten korrekt strukturiert und sinnvoll nutzbar sind (in Datenbanken/Ontologien)
  • verwendet um Qualität von Linked Data zu sichern & sicherzustellen, dass Systeme konsistente Daten verarbeiten

Beispiele

DBpedia

Eine semantische Datenbank, die Wikipedia-Inhalte strukturiert und verknüpft.

Google Knowledge Graph

Semantischen Technologien, die Daten aus verschiedenen Quellen (z. B. Wikipedia, Musikdatenbanken) verknüpfen

Wenn man nach „Taylor Swift“ sucht, zeigt Google eine Infobox mit biografischen Informationen, Alben, Tourdaten und verwandten Künstlern

Schema.org

Gemeinsame Initiative von Google, Bing und anderen, um strukturierte Daten für Suchmaschinen zu standardisieren.

Zukunftstaussicht

Weiterentwicklung der Standards

W3C arbeitet kontinuierlich an neuen Tools und Sprachen.

Mögliche Szenarien

  • Assistenten, die basierend auf semantischen Daten alltägliche Aufgaben übernehmen.
  • Personalisierte Dienste, die auf den Interessen und Vorlieben des Nutzers basieren.
  • Automatisierte Entscheidungsfindung in der Medizin, Finanzwesen, etc.
